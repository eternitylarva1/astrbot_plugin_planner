# Changelog

本文档记录 `astrbot_plugin_planner` 的功能更新。

## [1.2.0] - 2026-03-28

### Added
- 新增持久化事件流 `planning_events.json`：记录任务生命周期关键操作（创建/完成/取消/改期），支持从学习结果回溯原始事件。
- 新增 `/习惯 重建` 命令：根据事件流重新构建用户习惯档案，支持意外数据恢复。
- 新增 `/习惯 最近事件` 命令：查看最近的行为事件记录。
- 新增可配置的 AI 习惯规划能力：
  - `habit_planning_enabled`：是否启用习惯驱动自动规划
  - `habit_weight`：习惯建议权重（0-1）
  - `suggestion_count`：每次规划最大建议数
  - `max_daily_minutes`：每日习惯规划最大时长
  - `learning_confidence_threshold`：习惯置信度阈值
- 新增 `/计划反馈` 命令：用户可对规划结果给出即时反馈，影响后续推荐。
- 新增习惯数据清理与重置能力：
  - `/习惯 删除 时长 <任务名>`：删除特定任务的时长学习记录
  - `/习惯 删除 别名 <alias>`：删除别名学习记录
  - `/习惯 删除 时段 <任务名>`：删除时段学习记录
  - `/习惯 重置 全部`：清空所有学习数据（含二次确认机制）

### Changed
- 增强 `set_planner_config` 工具：支持动态调整习惯规划相关配置参数。
- 学习服务新增置信度评估与反馈感知的时段评分机制。
- 规划流程现在会遵守 `habit_planning_enabled`、`habit_weight`、`suggestion_count`、`max_daily_minutes` 等配置。

## [1.1.2] - 2026-03-27

### Added
- 新增 `planner_tool_contract` LLM 工具，返回“工具选型 + 参数规范”说明，帮助 LLM 稳定填参。
- 新增 `AI_PROJECT_SPEC.md`，沉淀用户需求与 AI 工具调用项目规范，便于后续其他 AI 接手。

### Changed
- 强化 `create_planner_task` 的参数错误反馈：缺少 `task_name/task_time/duration_minutes` 时返回结构化调用模板。
- 强化 `plan_with_ai` 文档中的工具选型规则，明确何时调用规划工具、何时直接创建任务。

## [1.1.1] - 2026-03-27

### Added
- 新增 `auto_plan_task` LLM 工具，专门承接“帮我安排/规划一下”这类口语化请求，默认支持直接创建任务。
- 新增 3 项可配置能力：`auto_plan_on_missing_time`、`avoid_past_time`、`ai_default_duration_minutes`。

### Changed
- 优化 `create_planner_task`：当缺少具体时间且配置允许时，自动调用 AI 规划逻辑补全可执行时间，不再只让用户补充。
- 优化 AI 规划语义清理，减少“这周安排一下…”等包装语进入任务名。

### Fixed
- 修复 `/ai规划` 可能返回过去时间的问题：开启 `avoid_past_time` 后自动顺延到未来时段。

## [1.1.0] - 2026-03-27

### Added
- 新增 `plan_with_ai` LLM 工具：支持在用户只给出模糊目标时，由 AI 自动拆分候选任务并给出建议时间/时长；支持 `auto_create=true` 一键落地创建。
- 新增 `/ai规划` 命令（别名：`智能规划`、`规划一下`、`自动规划`），可直接输入“这周安排一下……”这类自然表达获取计划建议。

### Changed
- 创建任务后改为自动学习“时长 + 时间段 + 别名”组合习惯，减少重复输入同类任务参数。
- 学习系统新增自动学习开关能力：支持 `/学习 自动开启`、`/学习 自动关闭`、`/学习 自动状态`。
- 帮助文案与 README 更新，补充 AI 规划与自动学习说明。

## [1.0.9] - 2026-03-27

### Changed
- 优化任务名提取策略：从冗长描述中优先提取核心任务语义片段，减少将“注意事项/提醒语”写入任务名的情况（如“参加宣讲会”）。
- 优化 `create_planner_task` 的任务名标准化：当输入为长句时进行二次提炼，尽量将时间信息与任务描述分离，提升 LLM 工具调用稳定性。
- 调整可视化页面为更适合手机竖屏：日视图/周视图均收窄默认宽度，并增强移动端样式适配（周视图在窄屏改为单列）。

### Fixed
- 修复部分场景下任务名中残留“月日/月号”等异常文本的问题。

## [1.0.8] - 2026-03-27

### Added
- 新增冲突检测与自动顺延能力：在任务创建入口统一接入时间冲突检查，并支持自动推荐下一个可用时间段。
- 新增时间解析单元测试，覆盖“中午”相关表达，降低自然语言时间解析回归风险。

### Changed
- 优化提醒调度加载策略：当检测到 AstrBot 原生调度能力可用时，跳过插件侧重复提醒重载，避免重复调度。
- 优化命令缺参提示：`/计划`、`/取消`、`/循环`、`/设置提醒`、`/设置超时` 在缺少参数时统一返回“用法 + 示例”。
- 规避帮助命令冲突：将插件帮助命令调整为 `规划帮助`（别名：`计划帮助`、`planner_help`、`使用说明`），避免触发 AstrBot 原生 `help`。
- 强化 LLM 任务流程与工具链路，补充任务管理相关工具能力，提升“创建-查看-完成-取消”闭环稳定性。

### Fixed
- 修复 `TimeParser` 对“中午”时间表达的解析错误（如“中午1点”），确保解析结果符合常规语义。

## [1.0.7] - 2026-03-25

### Added
- 新增 3 个 LLM 工具：`list_planner_tasks`、`complete_planner_task`、`cancel_planner_task`，支持通过自然语言驱动“查看/完成/取消”常用任务操作。

### Changed
- 加强会话隔离：任务列表、图表渲染、完成任务、取消任务均只操作当前会话创建的任务，避免跨会话误操作。
- 提升 LLM 工具可用性：在“信息完整时创建任务”的基础上，补齐任务管理链路，支持更多常见自然语言诉求闭环。

## [1.0.6] - 2026-03-25

### Changed
- 进一步放大日/周视图内容区占比：容器宽度提升到 `96vw` 且最小高度提升到 `92vh`，图表在图片中占据超过 3/4 面积。
- 全面放大字体并突出关键信息（任务名、时间、时长、统计值）为加粗样式，提升识别效率。
- 强化现代化视觉层次：更大圆角、阴影、信息徽章与分层排版。

## [1.0.5] - 2026-03-25

### Changed
- 放大图表主体在图片中的占比：减小页面留白并提高容器宽度，让日/周视图在最终图片中更突出。
- 提升可读性与对比度：调整标题、徽章、任务文本与周视图今日列样式，避免“背景好看但文字看不清”的问题。

## [1.0.4] - 2026-03-25

### Added
- 日视图支持多种可选样式：默认时间轴（竖向时间线 + 任务方块）、卡片、紧凑列表。
- `/图表` 支持通过关键词切换样式，例如：`/图表 卡片 本周`、`/图表 紧凑 今天`。

### Changed
- `render_daily_schedule` 新增 `style` 参数，用于按样式渲染日视图。

## [1.0.3] - 2026-03-25

### Added
- 新增 `/图表` 命令（别名：`可视化`、`查看图表`、`查看可视化`），支持主动查看可视化日程图。

### Changed
- 抽取并复用可视化渲染逻辑，`/任务` 命令与 `/图表` 命令统一使用同一套图表查询流程。
- 更新 `README.md`，补充 `/图表` 命令使用方式与示例。
